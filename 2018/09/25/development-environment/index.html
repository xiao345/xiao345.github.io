开发环境搭建

平时主要是做PHP开发，尤其是基于Laravel框架开发，但也不仅限于PHP开发，所以开发环境配置需要在满足PHP开发工作的前提下，还需要具备一定的灵活性，方便用于其他语言的开发，最好能做到可插拔。基于以上诉求，最终找到一下比较好的解决方案。

方案

Homestead + docker + phpstorm

思路

因为开发中经常需要安装各种工具，时间久了，就会导致环境混乱，所以我把所有的开发工具都放到虚拟机里，这样就等于我们有了一台专门用于开发的单机了。

但这还不够，我们在开发中，经常要用到很多的工具，每次都往这个虚拟机上装，时间久了也不好管理，但常用的工具还是应该要有的，那么哪些该装哪些不该装呢，因为平时主要是Laravel开发比较多，Laravel社区提供了一个很好的工具Homestead，相当好用，基本可以应付90%以上的开发需求了。

如果需要做其他语言的开发，还是建议不要直接往这个环境上装，这里docker就派上用场了，如要做go语言的开发，那就起一个go容器，在里面装各种工具，这样就能很好的管理你的开发环境。

简述

  1. windows 只装phpstorm等代码编辑器，用于日常代码编写
  2. 代码运行、调试等工具环境都跑在Homestead虚拟机上

相关工具

    1. virtualbox
    2. vagrant
  2. laradock
